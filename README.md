### v3.3 — 2025-05-16 (교재 학습 관리 예외 기능 버그 수정 + UI 개선)

#### 🐛 버그 수정 1: 예외 설정 — 반+교재 간 학생 데이터 독립성 문제

**문제**
- A반에서 AA교재에 "abc" 학생을 예외 등록한 뒤, 동일 A반의 BB교재에도 "abc" 학생을 등록하면 **AA교재에 등록된 "abc"의 예외 설정이 사라지는 버그**
- 원인: 예외 데이터를 `classId` 단위로 저장할 때 학생 이름이 flat key로 사용되어, 같은 이름의 학생이 여러 교재에 존재할 경우 덮어쓰기 발생

**수정 내용 (`booklib-app.js`)**

| 항목 | 변경 전 | 변경 후 |
|---|---|---|
| 저장 구조 | `{ 학생명: { bookId, items, ... } }` | `{ bookId: { 학생명: { items, ... } } }` |
| 저장 위치 | `hakwon10/exempts/{classId}` | 동일 (구조만 변경) |
| 교재 간 독립성 | ❌ 같은 이름 학생이 서로 덮어씀 | ✅ bookId를 1차 키로 완전 분리 |

**하위 호환성**: 기존 구버전 데이터 자동 마이그레이션 (`_migrateExemptsIfNeeded` 헬퍼 추가)

**영향 함수**
- `_migrateExemptsIfNeeded()` — 신규 추가, 구버전 데이터 자동 변환
- `_saveExemptMgr()` — 신구조로 저장 (해당 bookId 항목만 교체, 다른 교재 보존)
- `_loadExemptMgrList()` — bookId 기준 필터링으로 변경
- `_deleteExemptItem()` — 파라미터에 `bkId` 추가, 신구조 기반 삭제
- `_runBatchImport()` 내 exempts 로드 — 신구조 기반으로 수정
- `openCsvImportModal()` 내 exempts 로드 — bookId 필터 적용

#### ✨ 변경 4: 후처리 미수행 필터 (Post-Filter) — 신규 기능

xlsx 반영 후, 특정 학생의 특정 면제 항목을 **시각적으로만** 제거하는 기능입니다.

**특징**
- DB 저장 없음 — LocalStorage에 반+교재 단위로 저장
- 자동 적용 없음 — 사용자가 직접 "🔍 후처리" 버튼 → 적용 버튼으로 활성화
- 적용 후 "🔍 후처리 (N명)" 형태로 활성 상태 표시

**저장 키**: `bl_pf_{classId}_{bookId}`  
**데이터 구조**: `{ "학생명": ["게임", "테스트", ...] }`

**동작 방식**

| 상황 | 결과 |
|---|---|
| 해당 챕터의 미수행 세부항목이 모두 면제 항목 | 시각적으로 수행 처리 (✕ 사라짐) |
| 면제 항목 + 비면제 항목이 혼재 | 면제 항목 배지만 제거, 나머지 미수행 유지 |
| 세부항목 없이 미수행 처리된 경우 | 필터 미적용 (그대로 유지) |

**UI 위치**: 학습 현황 탭 → 통계 바 → 🔍 후처리 버튼

```
┌──────────────────────────────────────────────────┐
│  🔍 후처리 미수행 필터                    [✕]    │
│  A반 · AA교재 — xlsx 반영 후 시각적 면제 적용    │
│  💡 이 필터는 화면 표시에만 적용됩니다           │
├──────────────────────────────────────────────────│
│  새 필터 추가                                     │
│  [학생 이름 입력...▼]                            │
│  면제 항목: [암기][리콜][스펠][스피킹][게임][테스트]
│  [전체 선택]                  [＋ 추가]          │
├──────────────────────────────────────────────────│
│  👤 이채원  [게임] [테스트]               [🗑]  │
│  👤 홍길동  [암기]                        [🗑]  │
```

### v3.5 — 2025-05-16 (예외목록 반 자동 포커스 + 면제 항목 저장 버그 수정)

#### 🐛 버그 수정 1: 예외 설정 저장 시 면제 항목이 비어서 저장되는 문제

**원인** — `_addExemptRow` 내부 래퍼 `<div style="flex:1;position:relative">` 의 `position:relative`가 `_saveExemptMgr`의 셀렉터 `div[style*="position"]`에 **중복 매칭**되어, 내부 div로 처리 시 `items:[]`로 덮어쓰는 문제.

| 항목 | 변경 전 | 변경 후 |
|---|---|---|
| 행 셀렉터 | `div[style*="position"]` | `.em-row` 클래스 |
| 내부 래퍼 div | `style="flex:1;position:relative"` | `style="flex:1"` |
| 외부 행 마크 | 없음 | `row.className='em-row'` |

> ⚠️ 기존에 면제 항목이 비어 저장된 데이터는 예외 설정에서 **재저장** 시 정상 반영됩니다.

#### ✨ 기능 개선 2: 예외목록 — 현재 선택된 반으로 자동 포커스

학습 현황에서 반을 선택한 상태로 `📋 예외 목록`을 열면 슬라이더가 **해당 반 슬라이드로 자동 이동**합니다.

| 파일명 | 매칭 방식 |
